## 3. Чеклист скоринга

По каждому критерию: **Да (1)** или **Нет (0)**. Enterprise-first — **≥ 2 из 4** «Да».

| # | Критерий | Д (1) / Н (0) | Доказательства / комментарии |
|---|----------|---------------|------------------------------|
| 1 | **Платящие заказчики** — финансирование или требование от Enterprise в ближайшие два квартала | | |
| 2 | **Блокер сделки** — блокирует активную enterprise-закупку или продление | | |
| 3 | **Высокая стоимость поддержки** — релиз для всего CE создаст непропорциональную нагрузку до отработки паттернов | | |
| 4 | **Узкая аудитория CE** — маловероятно, что 80% развёртываний CE понадобится это за шесть месяцев | | |
| | **Итого (макс. 4)** | **/4** | |

## 4. Решение

Выберите **один** основной исход:

| Вариант | Когда выбирать |
|---------|----------------|
| **Community сейчас** | Итого 0–1, широкая ценность для CE, приемлемый риск |
| **Сначала Enterprise** | Итого ≥ 2, коммерческий сигнал оправдывает поэтапную поставку |
| **Бэклог** | Идея валидна, приоритет или ресурсы недостаточны |
| **Отказ** | Вне scope, противоречит принципам CE или неустойчиво |

**Основное решение:** `Community сейчас` · `Сначала Enterprise` · `Бэклог` · `Отказ`

## Краткая справка

| Балл | Типичное решение |
|------|------------------|
| 0–1 | Community сейчас (если в scope) |
| 2–4 | Сначала Enterprise (опубликовать квартал миграции в CE) |
| Любой + вне scope | Отказ |
| Валидно, нет ресурсов | Бэклог |

Элементы permanent Enterprise-only (поддержка, SLA, сертифицированные сборки, профуслуги) **не** проходят миграцию в CE по этому шаблону — укажите **Enterprise only** в разделе 5 без квартала CE.
