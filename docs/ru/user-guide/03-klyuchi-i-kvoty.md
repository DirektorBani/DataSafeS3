# 3. Ключи доступа, API-токены и квоты

[← Бакеты](02-dashbord-i-bakety.md) | [К содержанию](README.md) | Далее: [Безопасность →](04-bezopasnost-i-profil.md)

---

## Раздел Access

В меню слева выберите **Access**. Здесь две вкладки:

1. **Access Keys** — для программ и S3-совместимых приложений.
2. **API Tokens** — для REST API консоли.

---

## Access Keys (ключи S3)

Ключи нужны, чтобы подключать программы: AWS CLI, резервное копирование, свои скрипты.

### Системный ключ по умолчанию

При установке уже есть один ключ (из настроек сервера):

| Поле | Значение (локально) |
|------|---------------------|
| Access Key ID | `datasafe` |
| Secret Key | `datasafesecret` |
| Endpoint | `http://localhost:9000` |
| Region | `us-east-1` |

> Не публикуйте Secret Key. Это как пароль от хранилища.

### Создать свой ключ

1. Вкладка **Access Keys** → **Create key**.
2. Система покажет **Access Key ID** и **Secret Key** — **скопируйте Secret сразу**, потом его не покажут.
3. Сохраните ключ в надёжном месте (менеджер паролей).

### Удалить ключ

1. Найдите ключ в списке.
2. **Delete** → подтвердите.

Удалённый ключ сразу перестаёт работать.

---

## API Tokens (токены консоли)

Токены начинаются с `ds_` и дают доступ к **REST API** админ-консоли (не к S3 напрямую).

### Создать токен

1. Вкладка **API Tokens** → **Create token**.
2. Укажите имя, срок действия (или «без срока»), при необходимости — области (scopes).
3. Скопируйте токен — он показывается один раз.

### Когда нужен API Token

- автоматизация через HTTP (скрипты, CI/CD);
- интеграции, которые вызывают `/api/v1/...` с заголовком `Authorization: Bearer ds_...`.

Обычному пользователю для работы с файлами достаточно **Access Keys**.

---

## Usage (использование и квоты)

Раздел **Usage** в меню показывает, сколько места и объектов вы используете.

### Для обычного пользователя

- объём по каждому вашему бакету;
- диаграммы роста.

### Для администратора

- статистика **по всей системе**;
- сравнение с лимитами пользователей.

### Квоты (лимиты)

Администратор может задать лимит на пользователя:

| Параметр | Описание |
|----------|----------|
| Объём | Например 10 GB, 1 TB или «без лимита» |
| Число объектов | Максимальное количество файлов |

На **Обзоре** отображаются:

- **Использовано** — занято;
- **Квота** — лимит;
- **Остаток** — осталось.

При превышении лимита **новые загрузки** будут отклонены, пока администратор не увеличит квоту или вы не удалите файлы.

Квоты на отдельный бакет настраиваются в **Настройках** бакета (администратор).
