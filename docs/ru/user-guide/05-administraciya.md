# 5. Администрирование

[← Безопасность](04-bezopasnost-i-profil.md) | [К содержанию](README.md) | Далее: [Gateway →](06-gateway-i-minio.md)

> Раздел **Administration** виден только пользователям с ролью **administrator**.

---

## Users (пользователи)

**Administration → Users**

### Создать пользователя

1. **Create user**.
2. Заполните: логин, email, пароль, **роль** (user / operator / administrator).
3. Сохраните.

### Роли при создании

| Роль | Когда назначать |
|------|-----------------|
| user | Обычный доступ к своим данным |
| operator | Техподдержка, доступ ко всем бакетам без админ-настроек |
| administrator | Полное управление системой |

### Квота пользователя

1. В строке пользователя выберите **Set quota**.
2. Укажите объём (MB / GB / TB) или **Unlimited**.
3. При необходимости — лимит числа объектов.
4. Сохраните.

### Сброс пароля

**Reset password** → новый пароль → сообщите пользователю безопасным способом.

### Удаление

**Delete** — пользователь и его ключи перестанут работать. Бакеты могут остаться (зависит от владельца).

## Teams (команды)

**Команды** позволяют нескольким пользователям работать с **общими бакетами**.

Как это работает:

- бакет может быть привязан к **команде**;
- все участники команды видят и используют этот бакет;
- обычный пользователь **не** видит чужие личные бакеты.

> Отдельная страница «Teams» в консоли пока в разработке. Членство в командах и привязка бакетов настраиваются на уровне данных (миграция, API). Обратитесь к техническому администратору или см. [docs/context/](../context/).

## Tenants (арендаторы)

**Administration → Tenants**

Используется, когда одна установка Датасейф S3 обслуживает **несколько организаций** (multi-tenant):

1. **Create tenant** — имя организации.
2. **Add member** — выберите пользователя и роль в арендаторе:
   - **tenant_admin** — полный доступ к бакетам арендатора (чтение и запись);
   - **member** — чтение и запись объектов в бакетах арендатора;
   - **viewer** — только чтение (загрузка и удаление запрещены).
3. Бакеты с тем же `tenant_id`, что и арендатор, видны всем его участникам согласно роли.
4. Пользователи **вне** арендатора не видят чужие бакеты, даже если знают имя.

При создании бакета арендатор назначается автоматически по профилю владельца.

## Settings (настройки системы)

**Administration → Settings**

| Раздел | Что настраивается |
|--------|-------------------|
| **Trash** | Мягкое удаление, срок хранения в корзине (1–3650 дней) |
| **MFA** | Обязательная MFA для администраторов |
| **LDAP** | Вход через корпоративный каталог (Active Directory и т.д.) |
| **OIDC / SSO** | Вход через Keycloak, Authentik и др. |
| **Cluster** | Распределённый режим (см. [раздел 8](08-federation-i-cluster.md)) |

### Настройки отдельного бакета

Откройте бакет → **Settings**:

| Вкладка | Назначение |
|---------|------------|
| General | Описание, класс хранения (Hot/Warm/Cold) |
| Versioning | Включить версии объектов |
| Object Lock | WORM — запрет удаления до срока |
| Lifecycle | Автоудаление старых файлов / версий |
| Visibility | Тип хранения: **Private** (только авторизованные) или **Public read only** (анонимное чтение через S3) |
| Quotas | Лимит на бакет |

## Policies (политики доступа)

**Administration → Policies**

Политики описывают, **кто и что может делать** с бакетом (аналог IAM в AWS):

1. Выберите бакет.
2. Используйте **визуальный редактор** или вкладку **JSON**.
3. Сохраните.

Пример: разрешить только чтение (`GetObject`, `ListBucket`) определённому пользователю.

---

## Activity (журнал действий)

**Administration → Activity**

- кто и когда создал/удалил бакет или файл;
- входы, изменения настроек;
- фильтры по пользователю, действию, дате.

Полезно для аудита и расследования инцидентов.

## Webhooks (уведомления)

**Administration → Webhooks**

Отправка HTTP-запросов на ваш URL при событиях:

| Событие | Когда срабатывает |
|---------|-------------------|
| ObjectCreated | Загружен файл |
| ObjectDeleted | Удалён файл |
| BucketCreated / BucketDeleted | Создан / удалён бакет |
| UserCreated | Создан пользователь |

### Настроить webhook

1. **Create webhook**.
2. URL получателя, список событий, при необходимости — заголовки.
3. Сохраните.
4. В **Delivery log** смотрите успешные и неудачные доставки; можно **Retry**.
