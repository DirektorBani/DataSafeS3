# Что такое DataSafeS3?

**DataSafeS3 (Датасейф S3)** — платформа локального хранения для безопасного управления данными и политиками доступа.

Она объединяет **S3-совместимое объектное хранилище**, **веб-консоль**, **корпоративную аутентификацию** и **инструменты эксплуатации**, чтобы организация хранила данные, управляла доступом и наблюдала за использованием на своей инфраструктуре.

---

## Какую задачу решает DataSafeS3?

Командам нужно место для резервных копий, документов, медиа и blob-данных приложений, которое:

- остаётся на контролируемой инфраструктуре;
- интегрируется с корпоративной идентичностью (LDAP, SSO, MFA);
- даёт аудит и квоты;
- говорит на S3 API, который ожидают современные приложения.

DataSafeS3 закрывает эту потребность **единым продуктом**, а не набором скриптов вокруг «голого» диска.

## Для кого предназначен

| Аудитория | Типичная потребность |
|-----------|----------------------|
| **ИТ-администраторы** | Управление пользователями, тенантами, политиками и аудитом из одной консоли |
| **Platform-инженеры** | S3 для Kubernetes, CI/CD и внутренних сервисов |
| **ИБ / compliance** | Контур данных и логов on-prem или в выбранном регионе |
| **Подразделения / MSP** | Изоляция клиентов или отделов через мульти-тенантность |

## Ключевые возможности (ценность)

| Направление | Результат |
|-------------|-----------|
| **Безопасное хранение** | Данные с централизованным контролем доступа |
| **Интеграция идентичности** | Подключение существующих IdP и каталогов |
| **Governance** | Lifecycle, квоты, retention |
| **Аудит** | Журнал действий администраторов и пользователей |
| **Совместная работа** | Личное хранилище, совместный доступ к бакетам и контролируемые ссылки на файлы |
| **Наблюдаемость** | Мониторинг здоровья, ёмкости и репликации |

## Как поставляется

- **Один бинарник** `storage-server` — S3 API + Admin API + Gateway worker
- **Веб-консоль** — React UI с локализацией **EN / RU / DE / FR**
- **Docker Compose** — быстрый старт; опциональный [HA overlay](../../operations-guide/ru/reference-deployment-2node.md)
- **Helm chart** — production в Kubernetes (`values-production.yaml`, `values-ha.yaml`)
- **Метаданные** — BoltDB (по умолчанию) или PostgreSQL с маршрутизацией list на read replica
- **Артефакты релиза** — образы `ghcr.io/direktorbani/datasafe-*` (тег, напр. `v1.0.2`), SBOM, подписи cosign на тегах — проверка через `cosign verify` (см. [SECURITY.md](../../../SECURITY.md))
- **Интеграции** — STS, NATS, [extension hooks](../../../examples/extension-hook/README.md)
- **Файловое пространство** — home bucket, шаринг, опционально [sync CLI](../../../clients/README.md) — [статус](../../ru/specs/file-collaboration-status.md)
