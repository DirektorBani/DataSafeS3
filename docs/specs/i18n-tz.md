# Техническое задание: Мультиязычный интерфейс (i18n)

**Проект:** DataSafeS3 / Датасейф S3  
**Версия:** 1.0  
**Дата:** 2026-06-21

---

## Задача

Реализовать поддержку мультиязычного пользовательского интерфейса с возможностью переключения языка через селектор в правом верхнем углу приложения.

На текущем этапе необходимо поддержать только две локализации:

- Русский (RU)
- Английский (EN)

---

## Цель

Пользователь должен иметь возможность самостоятельно выбирать язык интерфейса.

Выбранная локализация должна применяться ко всему пользовательскому интерфейсу системы DataSafeS3.

---

## Переключатель языка

### Размещение

Разместить переключатель языка в правом верхнем углу интерфейса рядом с профилем пользователя.

### Вариант отображения

Компактный формат:

```text
RU | EN
```

или выпадающий список:

```text
Русский
English
```

### Поведение

При выборе языка:

- интерфейс переключается без перезагрузки страницы;
- пользователь остаётся в текущем разделе;
- все тексты обновляются мгновенно.

---

## Поддерживаемые локализации

На текущем этапе:

- `ru` — Русский
- `en` — English

Архитектура должна оставаться расширяемой для добавления новых языков в будущем, однако реализовывать дополнительные языки сейчас не требуется.

## Полное покрытие локализацией

Необходимо проверить и перевести все пользовательские тексты без исключений.

Запрещается оставлять смешанный интерфейс, когда часть элементов отображается на русском, а часть на английском.

После завершения работ интерфейс должен быть полностью локализован.

---

## Система переводов

Все текстовые строки необходимо вынести в централизованную систему локализации.

Запрещается хранить пользовательские тексты непосредственно в компонентах.

Пример:

```json
{
  "login.title": "Login",
  "login.password": "Password",
  "login.submit": "Sign In"
}
```

Структура:

```text
/locales
    /ru
        common.json
    /en
        common.json
```

---

## Определение языка

Приоритет выбора языка:

1. Язык, выбранный пользователем вручную (localStorage, ключ `datasafe.locale`).
2. Язык, сохранённый в профиле пользователя (поле `locale` в `GET /api/v1/me`).
3. Язык браузера (`navigator.language`).
4. Английский язык по умолчанию.

---

## Сохранение выбранной локали

После выбора языка настройка должна сохраняться:

- немедленно в `localStorage`;
- в профиле пользователя через `PATCH /api/v1/me/locale` с телом `{"locale": "en"|"ru"}` (для авторизованных пользователей).

При повторном входе пользователь должен видеть интерфейс на ранее выбранном языке.

### Backend API

| Метод | Путь | Описание |
|-------|------|----------|
| `GET` | `/api/v1/me` | Возвращает поле `locale` (если задано) |
| `PATCH` | `/api/v1/me/locale` | Обновляет локаль пользователя; допустимые значения: `en`, `ru` |

Поле `locale` хранится в таблице `users` (миграция `007_user_locale`).

## Локализация дат и времени

Необходимо локализовать:

- даты;
- время;
- названия месяцев;
- названия дней недели;
- относительное время.

Примеры:

```text
EN: June 21, 2026
RU: 21 июня 2026 г.
```

---

## Локализация чисел

Необходимо локализовать:

- формат чисел;
- размеры файлов;
- проценты;
- числовые показатели в интерфейсе.

Примеры:

```text
EN: 1,234.56
RU: 1 234,56
```

---

## Технические требования

- Использовать централизованную систему i18n (`react-i18next`).
- Поддержать динамическое переключение языка без перезагрузки страницы.
- Поддержать параметризованные переводы.
- Поддержать единственное и множественное число.
- Исключить хардкод текстов в компонентах.

---

## Дополнительная проверка

После реализации выполнить аудит интерфейса и убедиться, что локализованы:

- все страницы;
- все модальные окна;
- все уведомления;
- все формы;
- все ошибки API, отображаемые пользователю;
- все элементы первоначальной настройки;
- страница авторизации.

---

## Критерии приёмки

### Сценарий 1

Пользователь открывает DataSafeS3.

В правом верхнем углу отображается переключатель языка RU / EN.

### Сценарий 2

Пользователь выбирает русский язык.

Весь интерфейс отображается на русском языке без перезагрузки страницы.

### Сценарий 3

Пользователь выбирает английский язык.

Весь интерфейс отображается на английском языке без перезагрузки страницы.

### Сценарий 4

Пользователь выходит из системы и входит повторно.

Ранее выбранный язык сохраняется.

### Сценарий 5

В интерфейсе отсутствуют не локализованные элементы, хардкодные строки и смешение языков.
